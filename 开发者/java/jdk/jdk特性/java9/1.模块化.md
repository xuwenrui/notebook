### 1. 创建模块

假设你要创建一个名为`myApp`的应用程序，该程序依赖于一个名为`utils`的模块，用于提供一些通用工具方法。

#### 1.1 创建模块目录结构

首先，为每个模块创建一个单独的目录结构。例如，对于`utils`模块：

```
1utils/
2├── src/
3│   └── main/
4│       └── java/
5│           └── com/
6│               └── example/
7│                   └── utils/
8│                       └── UtilClass.java
9└── module-info.java
```

对于`myApp`模块：

```
1myApp/
2├── src/
3│   └── main/
4│       └── java/
5│           └── com/
6│               └── example/
7│                   └── myApp/
8│                       └── Main.java
9└── module-info.java
```

#### 1.2 编写`module-info.java`

在每个模块的根目录下创建`module-info.java`文件，定义模块名称、导出的包和所需的模块。

- `utils/module-info.java`:

Java

```
1module utils {
2    exports com.example.utils;
3}
```

- `myApp/module-info.java`:

Java

```
1module myApp {
2    requires utils;
3    opens com.example.myApp to utils;
4    exports com.example.myApp;
5}
```

这里，`myApp`模块显式地声明了对`utils`模块的依赖，并且开放了`com.example.myApp`包给`utils`模块访问（如果需要的话）。

### 2. 编译模块

使用`javac`命令编译模块，确保使用了正确的Java版本（至少是Java 9）：

Bash

```
1javac -d out --module-source-path src $(find src -name module-info.java)
```

这会将源代码编译到`out`目录下，并根据模块信息进行组织。

### 3. 打包模块

你可以使用`jar`工具为每个模块创建模块化的JAR文件：

Bash

```
1cd out
2jar --create --file=../../utils.jar --main-class=com.example.myApp.Main -C com.example.utils .
3jar --create --file=../../myApp.jar --main-class=com.example.myApp.Main -C com.example.myApp .
```

注意，上述命令中的`--main-class`参数应指向包含`main`方法的类，通常在应用模块中指定。

### 4. 运行模块化应用

最后，使用`java`命令运行模块化应用，指定主模块及其入口点：

Bash

```
1java --module-path . --module myApp/com.example.myApp.Main
```

这里，`--module-path`指定了模块的位置，`.` 表示当前目录；`--module`指定了包含主类的模块及其主类。
